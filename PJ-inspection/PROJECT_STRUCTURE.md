# 项目结构说明

## 文件组织

```
PJ-inspection/
├── main.py                 # 程序入口
├── main_window.py          # 主窗口界面
├── settings_dialog.py      # 设置对话框
├── config_manager.py       # 配置管理模块
├── review_manager.py       # 评审记录管理模块
├── api_client.py          # API客户端
├── file_scanner.py        # 文件扫描模块
├── requirements.txt       # Python依赖
├── run.sh                # 启动脚本（macOS/Linux）
├── README.md             # 项目说明
├── USAGE.md              # 使用指南
├── PROJECT_STRUCTURE.md  # 本文件
├── .gitignore           # Git忽略文件
├── config.json          # 配置文件（运行后生成）
└── reviews/             # 评审记录目录（运行后生成）
    ├── index.json       # 评审索引
    └── *.md            # 评审报告（Markdown格式）
```

## 模块说明

### 1. main.py
程序入口文件，负责：
- 创建Qt应用
- 初始化主窗口
- 启动事件循环

### 2. main_window.py
主窗口模块，包含：
- 界面布局和组件
- 文件树展示
- 文件内容预览
- 评审功能触发
- 历史记录展示
- 所有UI交互逻辑

**核心类**：
- `MainWindow`: 主窗口类
- `ReviewThread`: 评审线程（避免UI阻塞）

### 3. settings_dialog.py
设置对话框模块，负责：
- API配置界面
- 提示词模板编辑
- 配置验证和保存

**核心类**：
- `SettingsDialog`: 设置对话框类

### 4. config_manager.py
配置管理模块，负责：
- 加载和保存配置文件
- 提供配置项的读写接口
- 默认配置管理

**核心类**：
- `ConfigManager`: 配置管理器

**配置项**：
- `api_url`: API地址
- `api_key`: API密钥
- `prompt_template`: 提示词模板

### 5. review_manager.py
评审记录管理模块，负责：
- 保存评审结果到Markdown文件
- 管理评审索引
- 加载历史评审记录

**核心类**：
- `ReviewManager`: 评审管理器

**功能**：
- `save_review()`: 保存评审结果
- `get_review_content()`: 获取评审内容
- `get_all_reviews()`: 获取所有评审记录
- `delete_review()`: 删除评审记录

### 6. api_client.py
API客户端模块，负责：
- 与大模型API通信
- 处理API请求和响应
- 错误处理

**核心类**：
- `APIClient`: API客户端

**默认支持的API**：
- DeepSeek Chat Completions
- 兼容 OpenAI 风格接口的其他服务

### 7. file_scanner.py
文件扫描模块，负责：
- 扫描项目目录
- 生成文件树结构
- 读取代码文件内容
- 过滤不需要的文件和目录

**核心类**：
- `FileScanner`: 文件扫描器

**功能**：
- `scan_directory()`: 扫描目录并返回文件结构和代码内容
- `get_file_tree()`: 获取文件树（用于Qt展示）

**过滤规则**：
- 忽略隐藏文件和目录
- 忽略常见的依赖和构建目录
- 只读取指定扩展名的代码文件
- 限制单个文件大小（默认500KB）

## 数据流程

### 评审流程

```
用户点击"开始评审"
    ↓
检查API配置
    ↓
扫描项目文件 (FileScanner)
    ↓
生成文件结构和代码内容
    ↓
构建提示词 (使用模板)
    ↓
调用API (APIClient, 在后台线程)
    ↓
接收评审结果
    ↓
显示在界面上
    ↓
保存到本地 (ReviewManager)
    ↓
更新历史记录列表
```

### 配置流程

```
用户打开设置对话框
    ↓
加载当前配置 (ConfigManager)
    ↓
用户编辑配置
    ↓
验证配置有效性
    ↓
保存到config.json (ConfigManager)
    ↓
配置生效
```

### 历史记录流程

```
用户点击历史记录
    ↓
获取评审ID
    ↓
从reviews/目录加载MD文件 (ReviewManager)
    ↓
显示评审内容
```

## 技术栈

- **GUI框架**: PyQt5
- **HTTP客户端**: openai SDK (兼容 DeepSeek)
- **数据格式**: JSON, Markdown
- **Python版本**: 3.6+

## 扩展建议

如需扩展功能，可以考虑：

1. **支持更多API服务**
   - 扩展 `APIClient` 以接入其他兼容接口
   - 添加API类型选择

2. **代码高亮显示**
   - 使用 `QSyntaxHighlighter`
   - 集成 `Pygments` 库

3. **导出功能**
   - 导出为PDF
   - 导出为HTML
   - 批量导出

4. **对比功能**
   - 对比不同版本的评审结果
   - 显示差异

5. **多语言支持**
   - 使用Qt的国际化功能
   - 添加语言切换

6. **插件系统**
   - 自定义评审规则
   - 自定义文件过滤器

7. **实时评审**
   - 监听文件变化
   - 自动触发评审

## 注意事项

1. **API调用成本**：大项目可能产生较多API调用费用
2. **文件大小限制**：避免读取过大的文件（默认限制500KB）
3. **API超时**：大项目评审可能需要较长时间
4. **配置安全**：API密钥存储在本地，注意保护
5. **编码问题**：使用UTF-8编码，对非UTF-8文件使用ignore错误模式
