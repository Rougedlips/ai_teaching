### ✨ 主要改进

#### 1. 📋 右侧评审区域全高显示
- **之前**: 评审区域只占据右侧的1/3高度
- **现在**: 评审区域占据整个右侧高度，提供更大的阅读空间
- **布局比例**: 左侧内容区域 : 右侧评审区域 = 2 : 3

#### 2. 🃏 评审历史卡片化
- **卡片内容**: 每个卡片显示项目名称、日期和描述
- **卡片样式**: 现代化卡片设计，带边框和圆角
- **交互效果**: 
  - 悬停效果：浅蓝色背景
  - 选中效果：蓝色边框高亮
  - 滚动支持：固定最大高度，超出可滚动

#### 3. 🎯 当前评审高亮
- **自动高亮**: 新完成的评审自动高亮显示
- **手动选择**: 点击历史卡片自动加载并高亮
- **视觉反馈**: 选中卡片有明显的蓝色边框和背景色

#### 4. 📱 响应式布局
- **分割器**: 用户可调整左右区域比例
- **最小高度**: 历史区域最大高度200px，避免挤压主要内容
- **弹性空间**: 文件树和预览区域平分左侧空间

### 🔧 技术实现

#### 新增方法
- `highlight_review_item(review_id)`: 高亮指定评审记录
- 更新的 `load_review_history()`: 卡片式显示历史记录
- 更新的 `on_review_finished()`: 自动高亮新评审

#### 布局结构
- `main_splitter`: 主水平分割器 (左侧内容 vs 右侧评审)
- `left_splitter`: 左侧垂直分割器 (文件树 vs 文件预览)
- `history_list`: 卡片式历史记录列表
